# deepseek-v3.2 流式响应空文本问题

## 摘要

`deepseek-v3.2` 在测试环境模型发现中声明支持 `openai` 和 `anthropic`。非流式 `POST /v1/chat/completions` 与 `POST /v1/messages` 可以正常返回文本；但流式请求没有文本内容。

这会导致 opencode 等默认使用流式输出的生态工具表现为请求成功但无可见输出。

处理结论：该模型准备从平台模型列表下架，不再作为生态工具候选模型维护。

## 环境

- API base URL: `https://api.powertokens.ai/v1`
- 模型: `deepseek-v3.2`
- 上游渠道: `ali`
- endpoint 声明: `anthropic`, `openai`
- opencode 版本: `1.17.9`
- opencode provider: `@ai-sdk/openai-compatible`, `@ai-sdk/anthropic`

curl -sS \

| `deepseek-v3.2` | `ali` | `anthropic`, `openai` |

## opencode 复现现象

opencode 配置使用 OpenAI-compatible provider:

```json
{
  "provider": {
    "powertokens": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "powertokens",
      "options": {
        "baseURL": "https://api.powertokens.ai/v1",
        "apiKey": "{env:POWERTOKENS_API_KEY}"
      },
      "models": {
        "deepseek-v3.2": {
          "name": "deepseek-v3.2"
        }
      }
    }
  }
}
```

复现命令:

```bash
opencode run \
  --pure \
  --title audit-deepseek-v3.2 \
  --model powertokens/deepseek-v3.2 \
  --format json \
  "请只回复 OK，不要解释，不要调用工具。"
```

opencode 结果:

- 进程退出码: `0`
- 可见文本: 空
- `finishReason`: `other`
- token 事件: `input=11347`, `output=0`, `reasoning=0`, `cacheRead=0`

对照结果:

- `deepseek-v3-2-251201` 在同一 opencode 路径中返回 `OK`
- 多数其他文本模型在同一 opencode 路径中返回 `OK`

## 影响范围

- 直接非流式调用不受影响。
- 默认依赖 streaming 输出的生态工具会受影响，包括 opencode、OpenAI-compatible SDK streaming 调用、Anthropic SDK streaming 调用以及其他基于 SSE delta 渲染文本的客户端。
- 由于 HTTP 状态为 `200`，客户端通常不会按错误处理，只会展示空响应，排查成本较高。

## 初步判断

问题更像 `deepseek-v3.2` 的 streaming 适配问题，而不是模型发现、鉴权或非流式推理问题。

可能原因:

- 上游 `ali` DeepSeek 流式响应没有被正确转换为 OpenAI Chat Completions chunk 或 Anthropic Messages event。
- 网关只透出了 usage chunk，丢失了文本 delta chunk。
- 结束帧缺少 `finish_reason`，导致 opencode 记录为 `other`。

## 建议修复

1. 使用 request id `20260624015202175178308IUvpZHgW` 查询网关和上游日志，确认上游是否返回了文本 token。
2. 如果上游返回了文本，修复 OpenAI streaming 转换逻辑，确保输出 `choices[].delta.content`。
3. 如果上游本身不返回流式 token，可在模型声明中临时移除或标记该模型的流式 Chat Completions 支持边界，避免生态工具误用。
4. 增加回归测试: 同一 prompt 下同时断言 Chat Completions `stream:false` 有 `message.content`，Chat Completions `stream:true` 至少有一个 `delta.content` chunk，Messages `stream:true` 至少有一个 text content delta。

## 当前规避方案

- opencode 文档不要把 `deepseek-v3.2` 作为候选模型。
- 对 opencode 默认模型，优先使用已验证有可见 streaming 文本的模型，例如 `glm-5.2`。
- 下架完成前，如果历史业务仍请求 `deepseek-v3.2`，请优先迁移到其他模型。
